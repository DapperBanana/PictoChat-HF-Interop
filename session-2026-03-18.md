### Beacon frame format
- **Frame type**: Standard 802.11 Management Beacon (type 0, subtype 8 = 0x80)
- **BSSID OUI**: `00:09:BF` (Nintendo) — in the beacon's vendor IE; the outer BSSID MAC may use a newer Nintendo OUI
- **Supported rates**: **ONLY 1 Mbps + 2 Mbps** (bytes 0x82, 0x84) — critical identification signature
- **Short preamble** 802.11b
- **No SSID** element in beacon
- **Beacon interval**: ~105 TU (~107ms)
- **Beacons continuously** even when alone in a room ✅

### Vendor IE (Tag 0xDD) structure
```
dd 20                  Tag DDh, length 32
00 09 bf 00            Nintendo OUI + padding
0a 00                  Stepping offset
00 00                  LCD Video Sync
[4 bytes]              Fixed ID
[4 bytes]              Game ID (0x00000000 = PictoChat, non-zero = DS game)
[2 bytes]              Stream code (session identifier, changes per session)
08                     Payload length
01                     Beacon type: 01=PictoChat, 09=Empty, 0B=Multiboot
c0 00                  CMD data size
c0 00                  REPLY data size
48 23                  Fixed marker (always 0x2348)
[2 bytes]              Unknown
00                     Chatroom: 0=A, 1=B, 2=C, 3=D
01                     Connected user count (including host)
04 00                  Fixed marker (always 0x0004)
```

### Why we missed it in session 1
- Was searching for BSSID OUI `00:09:BF` but the DS Lite's outer MAC uses OUI `00:23:CC` (a newer Nintendo allocation)
- The `00:09:BF` OUI only appears **inside** the vendor IE payload, not as the BSSID
- The 1+2 Mbps-only rate signature was the right approach but previous captures were hopping channels and not staying on ch1 long enough

### Nintendo multicast addresses used in PictoChat
| Address | Direction |
|---|---|
| `03:09:bf:00:00:00` | Host → All clients |
| `03:09:bf:00:00:03` | Client acknowledgments |
| `03:09:bf:00:00:10` | Client → Host replies |

## Phase 1 Result: DS Beacon Captured and Decoded ✅

### Captured beacon
```
BSSID:    00:23:cc:f8:9e:3e
Channel:  1
Rates:    1 Mbps, 2 Mbps only
Signal:   -44 to -47 dBm (1.5 feet away, very strong)
Interval: ~107ms (105 TU)
```

### Vendor IE decode
```
OUI:           00:09:BF (Nintendo)
Game ID:       0x00000000 = PictoChat
Stream code:   0xAB38
Beacon type:   0x01 = PictoChat
Room:          0 = Room A  ✅ matches user report
User count:    1  (lone DS, no other participants yet)
Fixed markers: 0x2348 and 0x0004 both verified ✅
```

### Capture command that worked
Lock to ch1 (no hopping) and run:
```bash
sudo bash -c "nohup tcpdump -i wlan1 -w /tmp/nds.pcap --immediate-mode > /tmp/nds.log 2>&1 &"
```
Filter after the fact: look for `[1.0* 2.0* Mbit]` with no other rates, or grep for `dd 20 00 09 bf` in hex.
